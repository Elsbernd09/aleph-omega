# Formal Claim Registry

## Purpose

Phase 16 adds a formal claim registry to Project Aleph-Omega.

The purpose is to separate different types of claims:

- implemented claims
- tested finite claims
- stress-tested claims
- finite proof claims
- conjectural generalizations

This prevents the project from overstating what has been proven.

---

## Why This Matters

A serious mathematical project should distinguish:

1. what is implemented
2. what is tested
3. what is proved in a finite model
4. what is only conjectural
5. what is outside the current scope

The claim registry makes this distinction explicit.

---

## Claim Scope

The registry supports these claim scopes:

- finite model
- generated search space
- implementation level
- human proof sketch
- conjectural generalization

---

## Verification Levels

The registry supports these verification levels:

- unchecked
- documented
- tested by unit tests
- stress tested
- finite proof
- conjectural

---

## Current Registered Claims

The standard registry includes claims about:

- finite bridge distortion
- finite satisfaction preservation
- finite composition preservation
- no generated counterexamples in the finite search space
- conjectural general foundations relevance

---

## Correct Research Framing

Project Aleph-Omega can now honestly distinguish between finite verified claims and conjectural generalizations.

The strongest current claims are finite, computational, and model-bound.

The project should not claim that it has solved mathematical foundations or proved universal theorems about all logics.
