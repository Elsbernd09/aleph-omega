# Public Release README Notes

## Purpose

Phase 28A replaces the development-phase README with a cleaner public-facing README.

The old README is preserved as:

README_ARCHIVE.md

## Why This Matters

During development, the README accumulated many phase logs.

That is useful for internal history, but a public reviewer needs a clear front page explaining:

- what the project is,
- what it proves,
- what it does not prove,
- how to run it,
- where the Lean code lives,
- where the manuscript lives,
- how the Python layer relates to the Lean layer.

## Public Framing

The README frames Project Aleph-Omega as:

- a finite formal-methods research artifact,
- a semantic preservation laboratory,
- a Lean-supported quotient-category prototype,
- a serious mathematical-computation project.

## Claim Boundary

The README explicitly says the project is not yet:

- a universal theory of institutions,
- a proof about all logics,
- a full Mathlib Category instance,
- a complete Lean verification of every Python function,
- a solved open problem,
- a field-changing theorem.

## Correct Research Claim

Project Aleph-Omega now has a public-facing README that presents the project accurately, seriously, and without overclaiming.
