# Proof Obligation Tracker

## Purpose

Phase 16C adds a proof obligation tracker to Project Aleph-Omega.

A proof obligation records what a claim still needs in order to be stronger, clearer, or more responsibly framed.

---

## Obligation Statuses

The tracker uses these statuses:

- open
- satisfied
- out of scope
- conjectural

---

## Obligation Kinds

The tracker uses these kinds:

- formal proof
- unit test
- stress test
- documentation
- limitation
- generalization

---

## Why This Matters

The project now distinguishes between:

- what has evidence
- what has limitations
- what has finite proof or stress-test support
- what is still open
- what is only conjectural

This helps prevent overclaiming.

---

## Current Interpretation

A finite claim can have satisfied documentation and limitation obligations while still having an open obligation for stronger verification.

A conjectural claim can be responsibly marked as conjectural rather than treated as proven.

---

## Correct Research Claim

Project Aleph-Omega can now track proof obligations for its formal claims, making open verification work explicit.

This is not full machine proof.

It is a structured verification-management layer.
